## Risks carried into release

- liteparse pulls ~25 transitive crates + cmake C build (aws-lc-sys) when pdf on. Build-time cost only; gated behind default feature.
- libpdfium dlopen'd at runtime: missing dylib → runtime error in dev `cargo run` (CI hard-fails packaging if absent — correct guard).
- Scanned/image-only PDFs yield empty text (OCR off) — no explicit UI error beyond empty doc.
- Windows PDF permanently opted out (no ETA).
