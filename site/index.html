## Why rmdv

    

      For reading folders of .md files without spinning up Obsidian's vault,
      Typora's editor weight, or a static-site build. Open a folder. Read.

## Scriptable over IPC

    

      rmdv runs as a single instance behind a Unix socket. Any process can drive the
      open window from the command line: open a file, jump to a heading, switch view
      mode. Every command returns one line of JSON.
    

    
# open a file at a specific line
rmdv path/to/spec.md --line 42

# navigate the running instance
rmdv goto --section "API/Authentication"
rmdv mode mindmap
rmdv current                          # prints JSON state

# stateless: no window needed
rmdv list-sections spec.md | jq -r '.[].path'

    

      This makes rmdv easy to wire into editor plugins, shell scripts, or a coding agent
      such as Claude Code or Codex: write a doc, open it at the right heading, keep working.
    

  


  
  

    
## FAQ

    


      

        Is rmdv free and open source?
        
Yes. rmdv is MIT-licensed and free to use, modify, and distribute. Source code is at github.com/minchenlee/rmdv.

      


      

        What is the difference between document Mindmap and Full Mindmap?
        
Document Mindmap (⌘M) turns the current Markdown, JSON, or YAML document into a tree. Full Mindmap (⌘⇧M) is an app-level workspace navigator: it shows the folders and files in a workspace and lets you preview or open a selected file. The two surfaces are separate, so entering one does not overwrite the other.

      


      

        How is rmdv different from Obsidian?
        
rmdv is a read-focused viewer, not a note-linking knowledge base. It opens folders without vault setup and has no plugin ecosystem. Mermaid diagrams and LaTeX math render natively.

      


      

        How is rmdv different from Typora?
        
Typora is a paid ($15) editor-first app for macOS and Windows. rmdv is free, open source, read-focused, and also runs on Linux. It renders Mermaid diagrams and LaTeX math without a webview; Typora uses an embedded browser for rendering.

      


      

        Does rmdv work offline?
        
Yes. rmdv is a native desktop application with no cloud dependency. All rendering (Markdown, Mermaid diagrams, LaTeX math, mindmaps, PDF text extraction) happens locally, with no network requests at runtime.

      


      

        What diagram and math formats does rmdv support?
        
rmdv renders Mermaid diagrams (all major diagram types) and Graphviz DOT graphs natively in Rust. Block LaTeX math ($$…$$) is rendered by a pure-Rust layout engine, with no JavaScript and no KaTeX.

      


      

        How does rmdv work with Claude Code or other AI coding agents?
        
rmdv runs as a single instance with a Unix socket IPC interface. An agent uses the rmdv CLI to open files, jump to specific headings, or switch view modes programmatically. Every command returns one line of JSON. The agent can keep a spec file open in rmdv at the exact section the human needs while continuing its own work.

      


      

        How do I install rmdv?
        
On macOS, download the signed and notarized .dmg from github.com/minchenlee/rmdv/releases and drag rmdv to Applications. After opening the app, use the Install CLI command from the command palette to make rmdv available in Terminal. On Linux, download the x86_64 AppImage, launch it once, and use the same Install CLI command; it creates ~/.local/bin/rmdv, which must be on PATH. To build from source, install Rust 1.80 or newer, then run git clone https://github.com/minchenlee/rmdv && cd rmdv && cargo build --release. Windows builds use cargo build --release --no-default-features because PDF extraction is macOS/Linux only.

      


      

        Is rmdv faster than Electron-based markdown viewers?
        
rmdv is a native Rust application with no Electron, embedded webview, or JavaScript runtime. Historical v0.2.0 benchmarks on an Apple M2 MacBook Pro measured first view at about 150 ms and parsing a 10,000-line document at 8.1 ms; current builds may differ.
